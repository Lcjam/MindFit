Phase 1 — MVP
상담사 가입부터 채팅 상담·리뷰까지 핵심 루프를 최소한의 기능으로 구현합니다.
MVP

핵심 루프 검증

상담사 가입 → 관리자 승인 → 내담자 예약·결제 → 채팅 상담 → 리뷰까지 한 바퀴 돌아가는 최소 루프 구현

8주
Week 1 – 8
🏉 목표: 상담사 5명 이상 승인, 내담자 → 상담사 → 채팅 전체 플로우 E2E 작동, 결제 → 문진표 열람 연동 작동
W 1–2 프로젝트 기반 구축
  • 개발 환경 셋업 (Spring Boot 3.x + React 18 + MySQL 8 Docker Compose)
  • DB 스키마 확정 및 마이그레이션 (Flyway / Liquibase)
  • JWT 인증 구조 구현 (signup / login / refresh / email-verify)
  • 역할별 사용자 구조 (ROLE_CLIENT, ROLE_COUNSELOR, ROLE_ADMIN)
  • REST API 기본 구조 + 에러 응답 표준화
  • React 프로젝트 셋업 (Vite + TypeScript + Tailwind + React Router)
  • 공통 컴포넌트 (레이아웃, Protected Route, Axios 인터셉터)
W 3–4 상담사 온보딩 & 관리자 심사
  • 상담사 가입 폼 + 서류 업로드 (AWS S3 연동)
  • 관리자 심사 UI (서류 뷰어, 승인 / 거절 버튼)
  • 상담사 프로필 편집 화면 (승인 후 활성화)
  • 가용 시간 설정 (주간 반복 스케줄)
  • 상담사 목록 공개 API 구현
  • 내담자용 상담사 탐색 화면 (기본 리스트)
W 5–6 내담자 예약 · 문진표 · 결제
  • 상담사 탐색 필터 (우울 / 번아웃, 가격, 상담 방식)
  • 상담사 프로필 상세 페이지
  • 날짜·시간 선택 + 문진표 작성 UI
  • 결제 연동 (토스페이먼츠 SDK + Webhook 처리)
  • 결제 완료 시 채팅방 자동 생성 + 문진표 접근 권한 부여
  • 상담사 문진표 열람 기능
W 7–8 채팅 + 리뷰 + 통합 테스트
  • WebSocket 채팅 구현 (STOMP, Spring + SockJS 클라이언트)
  • 채팅 이력 DB 저장 및 조회 API
  • 세션 완료 처리 + 리뷰 작성 (익명 옵션 포함)
  • 예약 수락 / 거절 (상담사 액션)
  • 핵심 플로우 E2E 통합 테스트
  • 초기 배포 (AWS EC2 + RDS, HTTPS / SSL)
Phase 2 — v1.0
실제 운영 가능한 수준의 안정성과 사용자 경험을 완성합니다.
v1.0

운영 품질 확보

취소·환불 자동화, 이메일 알림, 정산 내역, 반응형 최적화로 실 서비스 운영 수준에 도달

+6주
Week 9 – 14
✅ 목표: 환불 자동화 100% 커버, 이메일 알림 모든 트리거 동작, 모바일 웹 주요 화면 반응형 완성
W 9–10 취소·환불 자동화 + 알림
  • 취소 정책 적용 (24시간 기준 전액 / 50% 환불)
  • 토스페이먼츠 부분 환불 API 연동
  • 이메일 알림 (Spring Mail): 상담사 신규 예약 수신
  • 이메일 알림: 내담자 예약 확정 / 거절 통보
  • 이메일 알림: 상담사 심사 승인 / 거절 통보
W 11–12 정산 내역 + 리뷰 고도화
  • 상담사 수익 정산 내역 UI (세션별 / 월별 조회)
  • 정산 계좌 등록 기능
  • 리뷰 목록 페이지 + 상담사 답글 기능
  • 리뷰 익명 표시 처리 완성 (닉네임 마스킹)
W 13–14 관리자 강화 + 반응형 최적화 + 성능 튜닝
  • 관리자 통계 대시보드 (가입자 수, 세션 수, 매출)
  • 분쟁 처리 기능 (강제 환불 실행)
  • 회원 관리 (계정 정지 / 탈퇴 처리)
  • 반응형 웹 (모바일 브라우저 주요 화면 최적화)
  • 검색 필터 고도화 (가격 범위, 상담 방식, 정렬)
  • 성능 최적화 (N+1 쿼리 해결, DB 인덱스 튜닝)

리스크 & 대응 전략
식별된 주요 리스크와 사전 대응 방안입니다.
리스크 가능성 영향도 대응 전략
상담사 자격 서류 위조
자격증·학위 위조로 인한 신뢰 손상 		중 높음 초기엔 관리자 수동 심사 철저히 진행. 스케일 이후 NICE 자격 검증 API 또는 학교·기관 직접 확인 프로세스 도입 검토.
WebSocket 채팅 확장성
동시 접속 급증 시 단일 서버 병목 		중 중간 MVP는 단일 서버 STOMP로 시작. 트래픽 증가 지표 모니터링 후 Redis Pub/Sub + 다중 서버 구조로 전환. 세션 수 기준 알람 설정.
결제 Webhook 누락
토스페이먼츠 Webhook 미수신으로 결제 상태 불일치 		저 높음 Webhook 재시도 로직 구현 + 주기적 결제 상태 폴링 배치 추가. 멱등성 키(Idempotency Key) 적용으로 중복 처리 방지.
의료법 경계 문제
"치료" 표현으로 인한 의료법 위반 리스크 		중 높음 전 서비스에서 "상담" 언어만 일관되게 사용, "치료·진단" 표현 금지. 법률 자문 1회 이상 필수 수행. 이용 약관에 서비스 범위 명시.
개인정보 유출
상담 내용·문진표 등 민감 데이터 노출 		저 높음 상담 데이터 및 문진표 암호화 저장. S3 프리사인드 URL로 파일 접근 제어. 정기적 보안 감사 및 OWASP Top 10 기준 점검.
